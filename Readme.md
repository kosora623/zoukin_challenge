2Dブラウザゲーム「20秒雑巾がけチャレンジ」仕様書
1. 概要
制限時間20秒以内に、画面上の「汚れ」を雑巾でどれだけ多く拭き取れるか（透明にできるか）を競う2Dカジュアルゲーム。

2. 技術スタック & アセット
開発環境: Vite + TypeScript (または vanilla JS)

描画手法: HTML5 Canvas API

プレイヤー（カーソル）画像: zoukin.png（雑巾の画像）

3. 画面構成 & UI
画面は1画面（シングルページ）で完結させる。

ヘッダー（UI領域）:

残り時間表示（例: 残り時間: 20.00秒 ※小数点2桁までリアルタイムカウント）

現在の達成度表示（例: お掃除率: 45.2%）

メイン（ゲーム領域）:

HTML5 <canvas> タグ（画面中央に配置）

リザルト（ポップアップ表示）:

残り時間が 0 になった瞬間に操作をロックし、最終スコア（％）とリトライボタンを表示する。

4. コアロジック & 実装要件
① 描画と「拭く」ギミック
Canvasの ctx.globalCompositeOperation = 'destination-out' を使用する。

下層に「背景（綺麗な床など）」、上層に「汚れ（不透明な色や画像）」を重ねる。

マウスドラッグまたはスマホのタッチ移動中、zoukin.png の位置に合わせて上層の汚れを消しゴムのように削り取る。

マウスカーソル自体は非表示（cursor: none）にし、zoukin.png がマウスの座標に追従して表示されるようにする。

② 面積（％）の計算ロジック（※破綻防止・軽量化）
ctx.getImageData() を使用してCanvasのアルファ値（透明度）を解析し、透明になったピクセル（値が0）の割合を計算する。

パフォーマンス対策: 毎フレーム（60fps）で解析すると高確率で処理が重くなり破綻するため、「0.1秒（100ms）に1回」のタイマー実行、または「マウス移動が一定ピクセル数を超えた時」のみ計算を実行する間引き処理を入れること。

③ ゲームループ & 時間管理
ゲーム開始と同時に20秒のカウントダウン（ミリ秒単位）を開始する。

requestAnimationFrame を用いて、残り時間とUI（お掃除率）をスムーズに更新する。

残り時間が 0 になった瞬間にイベントリスナー（Mousedown / Touchstart等）を完全に破棄（またはフラグで無効化）し、それ以上拭けないように制御する。
